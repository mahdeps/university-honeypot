# University Portal Honeypot — هاني بوت بوابة جامعية

هاني بوت ويب تفاعلي يحاكي بوابة خدمات الطلبة بواجهات مطابقة، مدعوم بقاعدة بيانات
طُعم ضخمة، طبقة حماية على مستوى المنظومة، ولوحة رصد (SOC) تلتقط **بصمة كاملة عن جهاز
كل من يدخل** على المصيدة. الهدف: جذب المهاجمين إلى نظام طُعم معزول وتسجيل وتصنيف كل
محاولة + استخبارات عن أجهزتهم — لإخراج تقرير دفاعي.

> ⚠️ **مشروع دفاعي/تعليمي.** الواجهات تحاكي شكل بوابة حقيقية لأغراض المحاكاة، لكنها
> **غير متصلة بأي نظام حقيقي ولا تحوي أي بيانات حقيقية** — كل الطلاب والعلامات والمالية
> مفبركة (`app/decoy_db.py`)، والبيانات الشخصية التي كانت بالواجهات الملتقطة **تمت
> إزالتها بالكامل** واستُبدلت بقالب محايد. شغّله في بيئة معزولة فقط.

---

## ما الذي يقدّمه

- **بوابة ديناميكية مطابقة** — تسجيل دخول + ~39 صفحة (علامات، مالية، جدول، خطة، ملف
  الطالب...). كل طالب يفوت يشوف **بياناته هو** مولّدة حيّاً من قاعدة البيانات.
- **قاعدة بيانات طُعم ضخمة** — ~15 ألف طالب، 346 ألف علامة، مالية، حسابات موظفين،
  مكتبة، رسائل... (~570 ألف صف). الدخول قابل للحقن عمداً (SQLi) كطُعم — لكن الاتصال
  **للقراءة فقط** فلا يقدر أي حقن يعدّل القاعدة.
- **مصادقة ASP.NET WebForms محاكاة** — `__VIEWSTATE`/`__EVENTVALIDATION`، كوكي
  `ASP.NET_SessionId`، وترويسات IIS — فالطلبات الآلية التي تتخطى النموذج تُرفض
  وتُرصد (`/error.aspx`).
- **طبقة حماية للمنظومة** (`app/security.py`) — راجع الأسفل.
- **لوحة رصد SOC + بصمة الأجهزة** (`/_monitor`) — راجع الأسفل.

## التشغيل

```powershell
cd university-honeypot
python -m venv .venv ; .\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python -m app.honeypot
```

- البوابة (واجهة المهاجم): http://127.0.0.1:8080/
- لوحة الرصد المحمية: http://127.0.0.1:8080/_monitor

أو عبر Docker (معزول): `docker compose up --build`

تجربة سريعة كطالب: أي حساب من القاعدة، مثلاً عبر حقن `' OR '1'='1' -- ` في خانة الدخول.

## لوحة الرصد + بصمة الجهاز (`/_monitor`)

محمية بـ **HTTP Basic auth**. بيانات الدخول الافتراضية:

```
user: mahde
pass: 123qweQWE@
```

كلمة المرور مخزّنة كـ **hash (PBKDF2)** فقط — هذا الملف آمن للنشر. بعد 5 محاولات
فاشلة من نفس الـIP يصير قفل 5 دقائق.

عند تحميل أي صفحة طُعم، سكربت (`app/intel.py`) يجمع من متصفح الزائر ويرسله لـ`/_collect`:
نظام التشغيل/المتصفح/المحرّك، كرت الشاشة (WebGL/GPU)، دقة الشاشة ونسبة البكسل،
**المنطقة الزمنية واللغة** (مؤشّر موقع بدون أي إذن)، عدد الأنوية والذاكرة، نوع الشبكة
والبطارية، بصمة Canvas والخطوط، **عناوين IP المحلية عبر WebRTC** (تكشف العنوان خلف
VPN/NAT)، علامات الأتمتة (`webdriver`/headless → **كشف بوت**)، و(اختياري) **GPS** إن
مُنح الإذن. اللوحة تعرض كل هذا لكل IP مع تغذية حيّة، رسوم، بحث، وتصدير JSON.

> هذه استطلاع سلبي لمن يتفاعل مع المصيدة (بيانات يكشفها المتصفح نفسه) — لأغراض
> الاستخبارات الدفاعية فقط.

## الإعداد والتحصين (متغيّرات البيئة)

| المتغيّر | الوظيفة | الافتراضي |
|---------|---------|-----------|
| `MONITOR_USER` / `MONITOR_PASS` | بيانات لوحة الرصد | `mahde` / (hash لـ`123qweQWE@`) |
| `MONITOR_PASS_HASH` | بدل كلمة المرور بـhash جاهز | — |
| `MONITOR_ALLOW_IPS` | IPs تتخطى المصادقة (مثلاً `127.0.0.1`) | فارغ = المصادقة دائماً |
| `HONEYPOT_SECRET` | مفتاح توقيع توكنات VIEWSTATE | عشوائي لكل تشغيل |
| `HONEYPOT_MAX_BODY` | حدّ حجم الطلب (DoS) | 262144 بايت |
| `HONEYPOT_RATE` / `HONEYPOT_RATE_WINDOW` | مانع الإغراق لكل IP | 240 / 10s |
| `HONEYPOT_HTTPS` | فعّله خلف TLS (يضيف Secure للكوكي) | — |

التحصينات المطبّقة: مصادقة + قفل تخمين على لوحة الرصد، مانع إغراق لكل IP، حدّ لحجم
الطلب، اتصال قاعدة بيانات للقراءة فقط، كوكي `HttpOnly`+`SameSite`, ترويسات أمان
كاملة، معالجات أخطاء لا تسرّب أي stack trace، وحدّ لذاكرة الجلسات.

> **استبدل `MONITOR_PASS` و`HONEYPOT_SECRET` قبل أي نشر فعلي.**

## الاستضافة على GitHub — مهم

GitHub يستضيف **الكود** (المستودع) ممتاز. لكن **GitHub Pages يخدم ملفات ثابتة فقط
ولا يشغّل Python/Flask** — يعني الهاني بوت **لن يعمل** عليه. لتشغيله فعلياً تحتاج
مستضيفاً يشغّل Python:

- **Render / Railway / PythonAnywhere / Fly.io** (مجاني/سهل لتطبيقات Flask)، أو
- **VPS أو Docker** على سيرفر تتحكم به.

وفي كل الأحوال: شغّله **معزولاً (مختبر/VM)**، ولا تنشر النسخة بشعار وقالب الجامعة على
دومين عام أو شبيه باسم الجامعة — لأن نسخة طبق الأصل لصفحة دخول مؤسسة حقيقية على
الإنترنت = صفحة تصيّد (phishing)، وهذا ممنوع. استخدمه لأغراض البحث الدفاعي فقط.

## خريطة الملفات

| الملف | الدور |
|------|------|
| `app/honeypot.py` | تطبيق Flask، المسارات، المصادقة، حقن البصمة، التقاط كل طلب |
| `app/security.py` | المصادقة، مانع الإغراق، حدود الطلب، الإعداد من البيئة |
| `app/intel.py` | سكربت بصمة جهاز الزائر (يُحقن بكل صفحة) |
| `app/dashboard.py` | تجميع الاستخبارات + لوحة `/_monitor` + APIs |
| `app/decoy_db.py` | مولّد قاعدة الطُعم الضخمة + الاستعلامات القابلة للحقن (read-only) |
| `app/dyn.py` | توليد الجداول الديناميكية (علامات/مالية/مساقات) لكل طالب |
| `app/detectors.py` | تصنيف الهجمات (SQLi/XSS/LFI/CMDi/SSTI/scanners) |
| `app/logger.py` | تسجيل كل حدث JSON في `logs/events.jsonl` |
| `app/portal/` | الصفحات المطابقة (منظّفة من البيانات الحقيقية) |
| `app/templates/` | الدخول، الخطأ، ولوحة الرصد |

`logs/` و`decoy.sqlite` متجاهلة بالـ`.gitignore` (تُولّد عند التشغيل وتحوي بيانات
الزوار) فلا تُرفع.
