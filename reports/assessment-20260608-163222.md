## Findings

### 1. SQL Injection — authentication bypass on /login

- **Severity:** Critical
- **Endpoint:** `/login`
- **Evidence:** Payload username="admin' OR '1'='1" redirected to /portal (logged in without valid creds).
- **Remediation:** Use parameterized queries / prepared statements. Never build SQL by string concatenation of user input. Add a generic auth failure message.

### 2. SQL Injection — UNION-based data extraction on /grades

- **Severity:** Critical
- **Endpoint:** `/grades?student_id=`
- **Evidence:** UNION SELECT against the students table returned credential-like rows in the response.
- **Remediation:** Parameterize queries; apply least-privilege DB accounts; do not expose other tables.

### 3. SQL Injection — error-based on /grades (student_id)

- **Severity:** High
- **Endpoint:** `/grades?student_id=`
- **Evidence:** Single quote in student_id produced a database error reflected to the client.
- **Remediation:** Parameterize the query. Disable verbose DB errors in responses; log them server-side.

### 4. Sensitive file exposed — Environment file (/.env)

- **Severity:** High
- **Endpoint:** `/.env`
- **Evidence:** HTTP 200 with content: 'APP_ENV=production\nAPP_KEY=base64:Zm9vYmFyZGVjb3lub3RyZWFsa2V5PT0=\nDB_CONNECTION=mysql\nDB_HOST=127.0.0.1\nDB_DATABASE=uni'
- **Remediation:** Block dotfiles and backups at the web-server level; never deploy secrets to webroot.

### 8. Missing security headers

- **Severity:** Low
- **Endpoint:** `/login`
- **Evidence:** Absent: content-security-policy, x-frame-options, x-content-type-options, strict-transport-security
- **Remediation:** Add a CSP to mitigate XSS. Add X-Frame-Options/CSP frame-ancestors to prevent clickjacking. Add 'X-Content-Type-Options: nosniff'. Add HSTS (when served over HTTPS).

### 9. Software version disclosure

- **Endpoint:** `/ (response headers)`
